# Feature: Internationalization (i18n)

## Overview

The app supports Arabic (ar, default) and English (en). A lightweight React context system handles translation, locale switching, and RTL/LTR direction.

## Implementation

Location: `frontend/src/i18n/`

- `context.tsx` — `I18nProvider`, `useTranslation()`, `registerTranslations()`
- `types.ts` — `Locale`, `Direction`, `TranslationMap`, `I18nContextType`, `LOCALE_CONFIG`
- `locales/ar.json` — Core Arabic translations (~90 keys)
- `locales/en.json` — Core English translations (~90 keys)

## API

```tsx
const { t, locale, setLocale, dir } = useTranslation();

t("name.greeting", { name: "أحمد" })  // → "أهلاً، أحمد"
setLocale("en")                         // switches to English
```

## Features

- **Locale persistence**: Stored in localStorage (`kalema_locale`)
- **Direction**: `dir` attribute on `<html>` toggles `rtl`/`ltr`
- **Fallback**: Missing keys fall back to Arabic, then return the key itself
- **Interpolation**: `{param}` syntax in translation values
- **Game translations**: Each game module registers its own translations via `registerTranslations()`

## Translation Key Organization

Core translations use dot-notation namespaces:
- `app.*` — App title, subtitle, version
- `common.*` — Shared UI labels (continue, cancel, confirm, etc.)
- `name.*` — Name entry
- `room.*` — Room management
- `create.*` / `join.*` — Create/join flows
- `admin.*` — Admin controls
- `game.*` — Game selection
- `error.*` — Error messages (mapped from error codes)
- `menu.*` — Menu items
- `notification.*` — Toast notifications

Game translations use game-prefixed namespaces:
- `impostor.*` — Impostor game translations

## Locale Switcher

Available in the hamburger menu (`Menu.tsx`). Toggles between Arabic and English labels.
