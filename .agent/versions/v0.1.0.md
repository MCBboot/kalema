## 6. Agent Coordination Rules

When multiple agents work on this project simultaneously:

1. **Read this file first** before starting any work
2. **Claim your task** by creating a file in `.agent/tasks/` (see task template below)
3. **Check `.agent/tasks/`** for active work by other agents to avoid conflicts
4. **Update SPEC.md** when adding new features or changing architecture
5. **Log changes** in `CHANGELOG.md` with the current version number

### Task File Format

Create `.agent/tasks/{agent-id}-{task-name}.md`:

```markdown
---
agent: {agent-id}
task: {short description}
status: in_progress | completed | blocked
started: {date}
files_touched:
  - path/to/file1
  - path/to/file2
---

## What I'm doing
{description}

## Blockers
{any blockers or none}
```

## How to Add New Requirements

When adding something new to this project:

1. Add a one-line entry to the **Features** table above (Section 3)
2. Create a detailed feature file in `.agent/features/{feature-name}.md` using the template
3. Add a changelog entry in `.agent/CHANGELOG.md`
4. If this is a major change, save a snapshot: copy this file to `.agent/versions/v{X.Y.Z}.md`
